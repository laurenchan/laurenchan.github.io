6.1 Using Circuitscape on Linux

To use Circuitscape on the Linux machine, we are going to use Julia which is computing language that is faster than the GUI (the GUI has also not been updated lately so this gives us some access to the more recent versions).

Files you'll need

You will need to prepare your input files first. You'll need an *.ini file and two *.asc files. Download this *.ini file and change the parameters to what you need (more on that below). You'll also need and ASCII file of your pts (export Raster to Ascii) and of your resistance or cost layer. Remember that the ASCIIs/rasters need to have the exact same dimensions. You can check this for the rasters in Properties>Source. You can alternatively look at the header (first few lines) of the ASCII in a text editor. (NOTE: If the ASCII isn't correct, you'll need to edit the rasters, not the ASCII.)

Put the ini file and the two asciis in a folder for just that run. Within that folder, make a new folder called "output". Make new folders and copy the ASCIIs for new runs.

INI file

The ini file holds all the info that Circuitscape will look to. For now, you will need to open it and change the path to the habitat_file and the path to the point_file. You'll need the path from wherever you're planning on running the program. I'm going to assume you'll be within the folder you created above so that you'll only need the filename.

Running on the Linux

You will need to run all of this on the Linux machine. Login with Splashtop or with Terminal/SFTP.

Move files

Upload your folder with infiles to whichever folder you want to run the analysis in - should be within your own directory.

Run using Screen

The Circuitscape run will take time. Longer than you want to be connected. You will use screen to start a session that you can keep running when you log off. See instructions for screen in Running analyses on Linux Tutorial

Briefly, on the Linux, you'll open Terminal, navigate to your folder using cd and then start a screen session by typing screen -S *your_runname*. For examplescreen -S laurenCS1`

Use Julia to start your run

Check to make sure you're in your correct directory. Type julia to start the Julia environment. At the prompt type:

using Circuitscape # loads the package into your environment
compute("path/to/config/file.ini") # should just be the name

Your run should start and you'll start to see calculations and timings. If you get an error, you'll need to read the error message and troubleshoot. If you need to exit Julia because of an error type ctrl-A-D. That should take you back to the normal bash prompt within your screen session. Type exit to leave/end that screen session.

If your run is running okay, use ctrl-A-D to exit the screen session and leave it running in the background. See the screen tutorial above on how to check back on your screen session.
